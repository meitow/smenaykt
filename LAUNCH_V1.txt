1. КОД: СТАТУС И ЛОКАЛЬНАЯ ПРОВЕРКА
================================================================================

1.1. Смержить ветки → ГОТОВО, всё в main.

1.2. Запуск локально:
     git checkout main && git pull origin main
     npm install
     cp .env.example .env
     # В .env: ADMIN_SECRET, MODERATOR_PHONES (ваш номер)
     npm run db:push
     npm run db:seed          ← только для локальной разработки!
     npm run build            ← обязательно: должен пройти без ошибок
     npm run dev

     Проверить коммит: git log -1  (должен быть a285e2b или новее)

1.3. Прогнать сценарии вручную (записать баги если есть):

     ЛЮДИ:
       - Заполнить профиль (имя + телефон)
       - Опубликовать личное задание (/post)
       - Взять чужое задание → чат → подтвердить выполнение с двух сторон
       - Оставить отзыв
       - Вкладка «Чаты» → открыть диалог → «Назад» в список
       - Потянуть вниз на главной / в чатах / в профиле → обновление
       - На вкладке «Создать» pull-to-refresh НЕ должен срабатывать
       - «Выйти» видна на любой вкладке профиля (не только «Контакты»)

     ПАРТНЁР:
       - /partner/login → ключ yktDemoPartner0000001 (после seed)
       - Опубликовать смену
       - В приложении увидеть смену с бейджем «Предприятие»
       - Назначить / дождаться исполнителя → чат

     МОДЕРАЦИЯ:
       - .env: ADMIN_SECRET + MODERATOR_PHONES=ваш_номер
       - Профиль → «Панель модерации» → панель открывается
       - «Выйти из панели» → снова из профиля → панель открывается
       - /admin?secret=... → скрыть задание, добавить бан
       - Вкладка «Документы» → одобрить / отклонить паспорт
       - Забаненный номер не может взять задание / писать в чат

3. ИНФРАСТРУКТУРА: ЧТО АРЕНДОВАТЬ — [ ] НЕ СДЕЛАНО
================================================================================

Минимальная схема для первых сотен пользователей:

  [Домен]      smenaykt.ru (или .рф)          ~200–500 ₽/год
  [VPS]        1–2 GB RAM, Ubuntu 22.04       ~300–1500 ₽/мес
  [PostgreSQL] managed или на том же VPS     0–500 ₽/мес
  [SSL]        Let's Encrypt через nginx        бесплатно

Провайдеры (примеры): Timeweb, Selectel, VK Cloud, Beget.
Альтернатива: Vercel (фронт) + Supabase (Postgres) — дороже, проще деплой.

Ориентир бюджета v1: 500–2000 ₽/мес + домен.

Что хранится где:
  PostgreSQL     — профили, задания, чаты, отзывы, баны, партнёры, документы (мета)
  Диск VPS       — аватарки: public/uploads/avatars/
  Диск VPS       — паспорта: private-uploads/ (не в git!)
  Позже v1.1     — S3/Object Storage для файлов

4. ДЕПЛОЙ НА VPS (ПОШАГОВО) — [ ] НЕ СДЕЛАНО
================================================================================

4.1. Подготовка сервера
     - Ubuntu 22.04, пользователь deploy, SSH-ключ
     - Установить: node 20 LTS, npm, nginx, certbot

4.2. PostgreSQL
     Вариант A — managed (Timeweb Cloud DB, Supabase):
       postgresql://user:pass@host:5432/dbname

     Вариант B — на VPS:
       sudo apt install postgresql
       createdb smenaykt + пользователь с паролем

4.3. Клонировать репозиторий
     git clone https://github.com/meitow/smenaykt.git
     cd smenaykt && git checkout main

4.4. Файл .env на ПРОДЕ (НЕ коммитить в git!)
     DATABASE_URL="postgresql://USER:PASS@HOST:5432/smenaykt"
     ADMIN_SECRET="случайная-длинная-строка-32+"
     MODERATOR_PHONES="+79XXXXXXXXX"
     NEXT_PUBLIC_SUPPORT_TELEGRAM="https://t.me/..."
     NEXT_PUBLIC_SUPPORT_EMAIL="support@smenaykt.ru"
     PRIVATE_STORAGE_BACKEND="local"
     PRIVATE_STORAGE_LOCAL_DIR="./private-uploads"
     NODE_ENV=production

     ⚠ НЕ использовать DATABASE_URL="file:./dev.db" на проде!

4.5. Сборка и миграция схемы
     npm ci
     npx prisma generate
     npx prisma db push
     npm run build

     ⚠ НЕ запускать npm run db:seed на проде!

4.6. Создать первых партнёров вручную (см. п. 2.2)

4.7. Запуск
     pm2 start npm --name smenaykt -- start
     pm2 save && pm2 startup

4.8. Nginx + HTTPS (certbot --nginx)

4.9. Папки для файлов
     mkdir -p public/uploads/avatars private-uploads
     chmod 755 public/uploads/avatars private-uploads
     Убедиться что процесс node может писать в обе папки.

4.10. Бэкапы
     - pg_dump по cron раз в сутки (БД — главное)
     - uploads/avatars + private-uploads — раз в неделю
